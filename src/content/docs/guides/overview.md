# Welcome to the RHCJC Documentation

This documentation serves as the central resource for understanding, maintaining, and replicating the design and functionality of the RHCJC website. Whether you're troubleshooting an issue, replicating a design, or reviewing plugin usage, this guide will help ensure consistency and clarity across all development work.

## 📚 Purpose of This Documentation

- To provide step-by-step guides for resolving common issues.
- To explain the reasoning behind design choices and plugin selections.
- To ensure anyone can replicate or maintain the website structure with minimal friction.

## 🧩 Plugins and Integrations

Understanding the plugins used is critical for site performance and troubleshooting. This documentation details:

- **What plugins are used** across the site.
- **Why each plugin was added**, including functionality and necessity.
- **Where each plugin is applied**, including specific pages or components.

You’ll also find a breakdown on the [Plugins](./plugins) page.

## 🧭 Types of Documentation

This site follows the [Diátaxis framework](https://diataxis.fr/) to ensure clarity and structure:

- **Tutorials**: Learn by doing with hands-on examples.
- **How-to Guides**: Step-by-step instructions for achieving specific tasks.
- **Reference**: Technical details and APIs.
- **Explanation**: Context and reasoning behind implementation choices.

## 🛠 When Things Go Wrong

Encountering an error or unexpected behavior?

Check the [Troubleshooting](./troubleshooting) page to find solutions to common problems, including plugin conflicts, deployment issues, and design mismatches.

## 🧱 Replicating Design

Need to rebuild or apply the same design elsewhere?
